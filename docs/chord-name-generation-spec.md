# コードネーム生成仕様

## 1. 基本方針

```text
最低音をルートとして固定する。
分数コードは使わない。
自然なコード推定よりも、構成音が読み取れる表記を優先する。
ただし、既存コード名の通例にはできるだけ従う。
```

---

## 2. 入力

```text
root = 最低音
notes = コード生成に使う音集合
```

`notes` は、次のいずれかである。

```text
正式コード用:
  notes = 構成音

候補コード用:
  notes = 構成音 + 候補音の一部
```

構成音は、必ず最低音を含む。

---

## 3. 分数コードは使わない

`C/E` のような分数コード表記は使わない。

最低音が E の場合は、必ず E をルートとしてコード名を生成する。

例：

```text
構成音: C E G
最低音: E
```

E基準では次のようになる。

```text
E G C = 1, ♭3, #5
```

出力：

```text
Em(#5)
```

---

## 4. 音程変換

最低音を `1` として、各音を半音差に変換する。

| 半音差 | 基本表記     |
| --: | -------- |
|   0 | 1        |
|   1 | ♭9       |
|   2 | 9        |
|   3 | ♭3       |
|   4 | 3        |
|   5 | 11       |
|   6 | ♭5 / #11 |
|   7 | 5        |
|   8 | #5 / ♭13 |
|   9 | 6 / 13   |
|  10 | ♭7       |
|  11 | 7        |

---

## 5. 表記の基本ルール

### 5.1 add は使わない

`add` 表記は使わず、括弧内に追加音を書く。

| 使わない表記  | 採用表記   |
| ------- | ------ |
| Cadd9   | C(9)   |
| Cadd♭9  | C(♭9)  |
| Cadd#11 | C(#11) |
| Cadd♭13 | C(♭13) |

ただし、`C6`、`Cm6` は通例を優先して使用する。

---

### 5.2 maj7 は M7 と書く

| 使わない表記 | 採用表記    |
| ------ | ------- |
| Cmaj7  | CM7     |
| CmMaj7 | CmM7    |
| Cmaj9  | CM7(9)  |
| CmMaj9 | CmM7(9) |

---

### 5.3 ♭5 は括弧に入れる

| 使わない表記 | 採用表記    |
| ------ | ------- |
| Cm7♭5  | Cm7(♭5) |
| C7♭5   | C7(♭5)  |
| CM7♭5  | CM7(♭5) |

---

## 6. 基本コード

### 6.1 三和音

| 度数      | 出力    |
| ------- | ----- |
| 1,3,5   | C     |
| 1,♭3,5  | Cm    |
| 1,♭3,♭5 | Cdim  |
| 1,3,#5  | Caug  |
| 1,4,5   | Csus4 |
| 1,2,5   | Csus2 |
| 1,5     | C5    |

---

### 6.2 四和音

| 度数         | 出力      |
| ---------- | ------- |
| 1,3,5,♭7   | C7      |
| 1,3,5,7    | CM7     |
| 1,♭3,5,♭7  | Cm7     |
| 1,♭3,5,7   | CmM7    |
| 1,♭3,♭5,♭7 | Cm7(♭5) |
| 1,♭3,♭5,6  | Cdim7   |
| 1,3,5,6    | C6      |
| 1,♭3,5,6   | Cm6     |

---

## 7. 9th 系

`C9`、`CM9`、`Cm9` のような省略表記は使わない。

7度と9度を分けて示す。

| 度数          | 出力      |
| ----------- | ------- |
| 1,3,5,♭7,9  | C7(9)   |
| 1,3,5,7,9   | CM7(9)  |
| 1,♭3,5,♭7,9 | Cm7(9)  |
| 1,♭3,5,7,9  | CmM7(9) |

---

## 8. 6 と 13 の扱い

### 8.1 7度がない場合

3度と5度があり、6度がある場合は、通例を優先して `6` を使う。

| 度数       | 出力  |
| -------- | --- |
| 1,3,5,6  | C6  |
| 1,♭3,5,6 | Cm6 |

### 8.2 7度がある場合

7度がある場合は、6度を `13` として括弧内に入れる。

| 度数           | 出力       |
| ------------ | -------- |
| 1,3,5,♭7,13  | C7(13)   |
| 1,3,5,7,13   | CM7(13)  |
| 1,♭3,5,♭7,13 | Cm7(13)  |
| 1,♭3,5,7,13  | CmM7(13) |

---

## 9. 半音6の扱い

半音6は、完全5度があるかどうかで表記を変える。

| 条件   | 表記  |
| ---- | --- |
| 5度あり | #11 |
| 5度なし | ♭5  |

---

## 10. 半音8の扱い

半音8は、完全5度があるかどうかで表記を変える。

| 条件   | 表記  |
| ---- | --- |
| 5度あり | ♭13 |
| 5度なし | #5  |

---

## 11. 3度がない場合

3度も短3度もない場合は、原則として `omit3` を付ける。

ただし、次の表記で3度なしが通例的に伝わる場合は、`omit3` を省略する。

| 度数      | 出力       |
| ------- | -------- |
| 1,5     | C5       |
| 1,4,5   | Csus4    |
| 1,2,5   | Csus2    |
| 1,2,4,5 | Csus4(9) |

---

## 12. 5度がない場合

完全5度、♭5、#5 のいずれもない場合は、原則として `omit5` を付ける。

| 度数      | 出力          |
| ------- | ----------- |
| 1,3     | C(omit5)    |
| 1,♭3    | Cm(omit5)   |
| 1,3,♭7  | C7(omit5)   |
| 1,♭3,♭7 | Cm7(omit5)  |
| 1,3,7   | CM7(omit5)  |
| 1,♭3,7  | CmM7(omit5) |
| 1,3,6   | C6(omit5)   |
| 1,♭3,6  | Cm6(omit5)  |

---

## 13. 括弧内の並び順

括弧内は、次の順番で固定する。

```text
♭5, #5, ♭9, 9, #9, 11, #11, ♭13, 13, omit3, omit5
```

---

## 14. 生成処理の順序

```text
1. 最低音をルートに固定する
2. notes をルートからの半音差セットに変換する
3. 3 / ♭3 の有無を判定する
4. 5 / ♭5 / #5 の有無を判定する
5. sus2 / sus4 / C5 にできるものを判定する
6. major / minor / dim / aug の骨格を作る
7. ♭7 / 7 / 6 を基本コード名に組み込む
8. 残りの音を括弧内に追加する
9. 必要に応じて omit3 / omit5 を追加する
10. 括弧内の順序を整える
```

---

## 16. 仕様の要約

```text
最低音をルートに固定し、分数コードは使わない。
add、maj7、Cm7♭5 のような表記は使わず、括弧・M7・omit を用いて構成音が読み取れる表記にする。
定番コードに一致する場合は C、Cm、C7、CM7、Cm7、C6、Cm6、Cdim、Caug、Csus4、Csus2、C5 を優先する。
不足音や変化音がある場合は、括弧内に ♭5 / #5 / ♭9 / 9 / #11 / ♭13 / omit3 / omit5 などを明示する。
```

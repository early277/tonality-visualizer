# コード表記表示仕様

## 1. 基本方針

Tonality Visualizer におけるコード表記は、次の2種類に分けて表示する。

1. 正式コード
2. 候補コード

正式コードは、構成音のみを使って算出する。  
候補コードは、構成音に候補音を加えた音集合から算出する。

---

## 2. 用語

| 用語 | 意味 |
|---|---|
| 最低音 | コード表記上のルートとして扱う音 |
| 構成音 | 正式コードの算出に必ず使う音 |
| ハイライト音 | 画面上でハイライトされている音 |
| 候補音 | ハイライト音から構成音を除いた音 |
| 正式コード | 構成音のみで算出したコード |
| 候補コード | 構成音に候補音の一部または全部を加えて算出したコード |

---

## 3. 前提条件

```text
ハイライト音は、構成音をすべて含む。
構成音は、最低音を必ず含む。
候補音 = ハイライト音 - 構成音
```

候補音の数は現在 0〜5 個とする。

候補音が最大 5 個の場合、候補音の組み合わせは最大 32 通りである。

```text
2^5 = 32
```

ただし、空集合は正式コードと同じになるため、候補コード生成では原則として除外する。

```text
候補コード数の最大 = 2^5 - 1 = 31
```

---

## 4. 表示順

表示順は次のとおりとする。

```text
1. 正式コード
2. 候補コード
```

正式コードは、スコアや自然さに関係なく必ず先頭に表示する。

候補コードは、候補順確認のため、当面はすべて表示する。

---

## 5. 正式コード

正式コードは、構成音のみから生成する。

```text
formalNotes = 構成音
formalCode = generateChord(formalNotes, bass)
```

正式コードには候補音を一切含めない。

例：

```text
最低音: C
構成音: C E G
候補音: D A

正式コード: C
```

---

## 6. 候補コード

候補コードは、構成音に候補音の任意の組み合わせを加えて生成する。

```text
candidateNotes = 構成音 + 候補音の一部
candidateCode = generateChord(candidateNotes, bass)
```

候補音が `D, A` の場合、候補コード生成に使う音集合は次のようになる。

```text
構成音 + D
構成音 + A
構成音 + D + A
```

---

## 7. 候補コードの生成手順

```text
1. formalNotes = 構成音
2. optionalNotes = ハイライト音 - 構成音
3. formalCode = generateChord(formalNotes, bass)
4. optionalNotes の全非空部分集合を作る
5. 各部分集合について、構成音と結合する
6. generateChord() でコード名を生成する
7. formalCode と同一表記の候補コードは除外する
8. 候補コードをスコア順に並べる
9. 正式コードを先頭に表示する
10. 候補コードをすべて表示する
```

---

## 8. 候補コードの重複処理

候補音の組み合わせが異なっても、同じコード表記になる可能性がある。

その場合は、同一コード名を1つにまとめる。

```text
同じコード名は1つに統合する。
統合時は、使用候補音が多いもの、またはスコアが高いものを代表とする。
```

---

## 9. 候補コードの並び順

候補コードはスコア順に並べる。

正式コードはスコア対象外とし、常に先頭に固定する。

候補コードのスコアリングでは、次の要素を考慮する。

```text
1. omit3 / omit5 が減るものを優先
2. 定番コードになるものを優先
3. 6, ♭7, 7, 9, 11, 13 など自然な拡張音を優先
4. 使用候補音が多すぎないものを優先
5. 変化音が少ないものを優先
6. 表記が短いものを優先
```

---

## 10. スコアリング方針

### 加点例

| 条件                               | 加点目安 |
| -------------------------------- | ---: |
| 定番コードに完全一致                       | +100 |
| omit3 が解消される                     |  +60 |
| omit5 が解消される                     |  +60 |
| 候補音を1つ採用                         |  +15 |
| 候補音が 6 / ♭7 / 7                  |  +20 |
| 候補音が 9 / 11 / 13                 |  +15 |
| 候補音により C6 / C7 / CM7 / Cm7 などになる |  +30 |

### 減点例

| 条件                  | 減点目安 |
| ------------------- | ---: |
| omit3 を含む           |  -60 |
| omit5 を含む           |  -40 |
| 括弧内の項目1つごと          |   -8 |
| ♭9 / #9 / #11 / ♭13 |  -10 |
| ♭5 / #5             |  -15 |
| 括弧内が4項目以上           |  -20 |
| 表記が過度に長い            |  -30 |

---

## 11. 表示例

### 例1

```text
最低音: C
構成音: C E G
候補音: D A
```

表示：

```text
正式コード: C

候補コード:
1. C6(9)    使用候補音: D, A
2. C6       使用候補音: A
3. C(9)     使用候補音: D
```

---

### 例2

```text
最低音: C
構成音: C E
候補音: G
```

表示：

```text
正式コード: C(omit5)

候補コード:
1. C        使用候補音: G
```

---

### 例3

```text
最低音: C
構成音: C G
候補音: E B♭
```

表示：

```text
正式コード: C5

候補コード:
1. C7       使用候補音: E, B♭
2. C        使用候補音: E
3. C5(♭7)   使用候補音: B♭
```

---

## 12. 表示形式

当面は、候補順確認のため候補コードをすべて表示する。

省スペース表示が必要になった場合は、次のような形式も検討する。

```text
C
候補: C6(9) / C6 / C(9)
```

詳細表示では、使用した候補音も表示する。

```text
正式コード: C

候補コード:
1. C6(9)    使用候補音: D, A
2. C6       使用候補音: A
3. C(9)     使用候補音: D
```

---

## 13. 確定仕様

```text
構成音のみで算出したコードを正式コードとして先頭に固定する。
候補音を考慮したコードは、候補コードとして2番目以降に表示する。
候補コードは、候補順確認のため当面すべて表示する。
候補コード内のみスコアリングで並び替える。
```
